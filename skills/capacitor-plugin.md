# Skill: Add a Capacitor plugin

**Use when:** the app needs a native capability that a Capacitor plugin provides.

## Required reading

- [rules/11 — Capacitor and native boundaries](../rules/11-capacitor-native-boundaries.md) — plugin
  owners and platform facades.
- [ADR 0006 — Capacitor boundary](../architecture/adrs/0006-capacitor-boundary.md) — one owner per
  plugin, composed by `src/platform`.
- [docs/eslint/no-raw-capacitor-imports](../docs/eslint/no-raw-capacitor-imports.md) — the rule that
  fires the moment you forget.
- [capacitor-app.facade.ts](../src/packages/capacitor-app/capacitor-app.facade.ts) — the owner
  pattern including listener cleanup.
- [context/native-capability-map](../context/native-capability-map.md) — what is installed and why.

## Preconditions

- [ ] The capability cannot be met by an installed plugin. Twelve are already owned — app, browser,
      device, haptics, keyboard, network, preferences, share, splash-screen, status-bar, plus
      `@capacitor/core` and secure storage.
- [ ] If the plugin needs a runtime permission, **stop**: permission-heavy plugins are deliberately
      not installed and need a security review first — see [permission-flow](permission-flow.md).
- [ ] You know the web behavior: real support, a no-op shim, or unavailable.

## Files

```text
src/packages/capacitor-<name>/capacitor-<name>.facade.ts   the only importer of the plugin
src/packages/capacitor-<name>/index.ts                     public surface
src/platform/<capability>/<capability>.facade.ts           composes the owner into app semantics
src/platform/index.ts                                      edit: export the platform facade
eslint/package-ownership.config.mjs                        edit: registry entry — mandatory
package.json                                               edit: exact pinned version
android/ + ios/                                            regenerated by cap sync
```

## Steps

1. Install pinned at the Capacitor major already in use (`8.x`): `npm i @capacitor/<name>@8.0.0`.
2. **Register the owner** in `eslint/package-ownership.config.mjs`:

   ```js
   {
     vendor: '@capacitor/<name>',
     owner: '@/packages/capacitor-<name>',
     ownerDirs: ['src/packages/capacitor-<name>'],
   }
   ```

   Without it `npm run quality:package-ownership` fails with
   `dependency "@capacitor/<name>" has no owner`, and `architecture/no-raw-package-imports` plus
   `architecture/no-raw-capacitor-imports` fire on every import — your facade included.

3. **Write the owner facade.** One file, importing the plugin, exporting app-shaped functions.
   `capacitor-app.facade.ts` is the model: `getLaunchUrl(): Promise<string | null>` unwraps
   `App.getLaunchUrl()` so callers never see the plugin's result object.
4. **Listeners return a cleanup, always.** Copy the `toCleanup` shape: `App.addListener(...)` yields
   a `Promise<PluginListenerHandle>`, so wrap it in
   `() => { void handlePromise.then((handle) => handle.remove()); }` and return that from
   `subscribeToX`. `architecture/require-native-listener-cleanup` and
   `architecture/no-floating-native-listeners` enforce the calling side — see
   [native-listener](native-listener.md).
5. **`index.ts` re-exports only** the facade's functions. `quality:exports` rejects anything else.
6. **Compose in `src/platform`.** Owners are thin; app semantics live one layer up.
   `src/platform/deep-links/deep-link.listener.ts` uses `@/packages/capacitor-app` and adds the
   runtime gate and the allowlist. Modules import `@/platform`, never `@/packages/capacitor-*`.
7. **Gate on the runtime** whenever the web shim lies. `isNativeRuntime()` from
   `src/platform/runtime/runtime.facade.ts` is the check; the deep-link listener's doc comment is
   the worked example of why.
8. **Sync the native projects:** `npm run cap:sync` (builds, then `cap sync`). Then
   `npm run cap:sync:check` proves android/ and ios/ are not drifting and the appId still matches
   `APP_IDENTITY`.
9. Add the plugin's config block to `capacitor.config.ts` only if it needs one — the existing
   `SplashScreen`, `Keyboard` and `StatusBar` entries show the shape. Never add `server.url`.

## Tests

- `capacitor-<name>.facade.test.ts` — mock the plugin (`vi.mock('@capacitor/<name>')`) and prove the
  unwrapping, the argument passing, and that `subscribeToX` returns a cleanup that calls `remove()`.
- `src/platform/<capability>/<capability>.facade.test.ts` — prove the composition and the native/web
  branch, mocking `isNativeRuntime`.
- Native reality is not covered by Vitest. Verify on a device — see [native-test](native-test.md).
- Run: `npx vitest run --project unit src/packages/capacitor-<name> src/platform/<capability>`.

## Security / accessibility / native considerations

- Any plugin reading user data (contacts, location, camera, clipboard) is the **critical** lane and
  needs a security review before install — see [security-review](security-review.md).
- New native dependencies change the Android manifest and iOS Info.plist; review the generated diff
  rather than committing it blind.
- iOS build verification is **UNVERIFIED off macOS** — `npm run ios:verify` says so and exits 0.
  Report it that way; do not imply an iOS pass you did not observe.

## Validation

```bash
npm run quality:package-ownership
npx vitest run --project unit src/packages/capacitor-<name>
npm run cap:sync && npm run cap:sync:check
npm run android:verify
npm run ios:verify
```

## Forbidden shortcuts

- `import { Camera } from '@capacitor/camera'` inside a module —
  `architecture/no-raw-capacitor-imports`.
- Installing the plugin and skipping the registry entry — `npm run quality:package-ownership` fails,
  and no amount of facade discipline fixes it.
- Calling `Capacitor.isNativePlatform()` in a module — `@capacitor/core` is owned by
  `src/platform/runtime`; use `isNativeRuntime()` from `@/platform`.
- Committing native project changes without `cap:sync:check` — drift ships as a broken build.

## Definition of done

- [ ] The plugin is pinned, registered, and imported only inside its owner package.
- [ ] Every `subscribeTo*` returns a working cleanup.
- [ ] A platform facade owns the app semantics and the runtime gate; modules import `@/platform`.
- [ ] `cap:sync:check` is clean and `android:verify` passes; iOS is reported honestly.
- [ ] `context/native-capability-map.md` documents the capability.
