# Skill: Run final validation

**Use when:** a change is finished and you are about to claim it is done.

## Required reading

- [rules/30 ‚Äî Release gates](../rules/30-release-gates.md) ‚Äî what each gate covers and when it is
  mandatory.
- [rules/00 ‚Äî Non-negotiable rules](../rules/00-non-negotiable-rules.md) ‚Äî the hard invariants the
  gates exist to defend.
- [package.json](../package.json) ‚Äî the composite scripts; read them rather than trusting memory.
- [verify-ios-environment.mjs](../scripts/native/verify-ios-environment.mjs) ‚Äî why an exit code of 0
  is not always a pass.
- [.ai/risk-lanes.md](../.ai/risk-lanes.md) ‚Äî routine, standard, critical, and the gates each
  demands.

## Preconditions

- [ ] The work is complete. Validation is a check, not a debugging loop.
- [ ] The risk lane is classified: **routine** (localized UI or docs), **standard** (behavior, deps,
      API, storage, routing, native, ownership), **critical** (auth, tokens, permissions, deep
      links, secure storage, native release, migration, secrets, privacy).
- [ ] Canonical docs are updated already ‚Äî `quality:docs` and `knowledge:validate` will fail
      otherwise.

## Files

```text
(validation writes nothing; it only reads)
package.json                 quality ‚Üí validate:web ‚Üí validate:native ‚Üí validate
.ai/**                       regenerated by knowledge:build, never hand-edited
```

## Steps

1. **Lane first, gates second.** Routine: targeted tests, lint, typecheck. Standard: add coverage,
   architecture, ownership, build. Critical: add integration, contract, `security:audit`,
   `security:secrets`, and E2E. Escalate when unsure; never de-escalate to save time.
2. **`npm run quality`** is the spine, in this order: `format:check`, `lint`, `typecheck`,
   `typecheck:toolchain`, `quality:architecture-rules`, `test:coverage`, `test:coverage:per-file`,
   `build`, `quality:architecture`, `quality:package-ownership`, `quality:dead-code`,
   `quality:circular`, `quality:duplicates`, `quality:exports`, `quality:locales`, `quality:docs`,
   `quality:agent-docs`. It is `&&`-chained, so it stops at the first failure ‚Äî the later gates are
   unrun, not passing.
3. **`npm run validate:web`** = `quality` plus `test:e2e`, `test:a11y`, `test:visual`,
   `security:audit`, `security:secrets`, `security:scan`, then `knowledge:build`,
   `knowledge:validate`, `knowledge:benchmark`.
4. **`npm run validate:native`** = `cap:sync`, `cap:sync:check`, `android:verify`, `ios:verify`.
5. **`npm run validate`** = `validate:web && validate:native`. This is the full sequence.
6. **Regenerate the knowledge plane** after any canonical doc change: `npm run knowledge:build`,
   then `knowledge:validate`. `.ai/` is generated (ADR 0015) ‚Äî every file there carries a
   DO-NOT-EDIT banner; editing it by hand is how the two planes diverge.
7. **Read the iOS line.** Off macOS, `ios:verify` prints
   `iOS build verification UNVERIFIED on this platform` and **exits 0** ‚Äî deliberately, because
   skipping honestly beats faking a pass. `validate:native` therefore "succeeds" without ever
   compiling iOS. Report: **iOS UNVERIFIED ‚Äî requires macOS with Xcode.**
8. **Note what could not run.** `security:scan` and `security:secrets` need Trivy; `android:verify`
   needs Java. If a tool is absent, the gate did not run ‚Äî say so.
9. **Report only what you executed.** Name the commands, their outcome, and the gaps. "Tests pass"
   is not a report; "`npm run quality` passed; `test:e2e` passed; iOS UNVERIFIED off macOS;
   `security:scan` not run (Trivy unavailable)" is.

## Tests

- The whole gate set is the test. What each proves, briefly: `test:coverage:per-file` names every
  file under threshold; `quality:architecture` checks layers, indexes and READMEs;
  `quality:package-ownership` checks vendor owners; `quality:exports` checks index purity;
  `quality:locales` checks en/ar parity; `quality:docs` checks every relative link and every rule
  doc; `quality:agent-docs` checks the agent entrypoints.
- While iterating, run the narrow gate ‚Äî the composite is for the end.
- Run: `npm run validate` for the full sequence.

## Security / accessibility / native considerations

- Critical-lane work is not done until `security:audit`, `security:secrets` and the E2E suite have
  all actually run.
- A green `test:a11y` means "no axe violations at the WCAG 2.2 AA tags" ‚Äî not "accessible". Keyboard
  order and screen-reader sense are manual; report them as manual.
- Device behavior is never covered by these gates ‚Äî see [native-test](native-test.md).

## Documentation delta

- None from validation itself. If `quality:docs` or `knowledge:validate` fails, the delta was owed
  earlier and is now overdue.

## Validation

```bash
npm run quality
npm run validate:web
npm run validate:native
npm run validate
```

## Forbidden shortcuts

- **Claiming a gate you did not run.** This is the one unforgivable failure here: it corrupts every
  future decision that trusts the report.
- Reading `&&`-chain success as "all gates passed" when it stopped early ‚Äî the first failure means
  everything after it is unrun.
- Reporting `validate:native` as green off macOS ‚Äî `ios:verify` exits 0 while printing UNVERIFIED.
- `--max-warnings` tweaks, `.skip`, or `test.only` to get through ‚Äî CI runs the real scripts.
- Hand-editing `.ai/` to make `knowledge:validate` pass ‚Äî it is generated; fix the canonical source
  and rebuild.
- Calling a green suite "verified on device". Different bundle, different platform, different bugs.

## Definition of done

- [ ] The risk lane was classified and the matching gates were run.
- [ ] `npm run quality` passed to its last gate, not just to its first failure.
- [ ] Web validation ran; native validation ran wherever it could.
- [ ] `.ai/` was rebuilt from canonical sources, not edited.
- [ ] The report names each command, its real outcome, and every gap ‚Äî iOS UNVERIFIED off macOS
      included.
