# 24 — Documentation

> Authority: normative. Conflicts resolve in favor of ADRs and the security policy.

## Mandatory

- MUST ship a `README.md` with every module under `src/modules/<name>/`, covering its public
  surface, its anatomy, its invariants, and where its tests live.
- MUST document every architecture ESLint rule at `docs/eslint/<rule>.md` — the docs gate asserts
  one file per rule name in the plugin.
- MUST update the documentation in the same commit as the behaviour: a surface change with a stale
  README is an incomplete change, not a follow-up.
- MUST keep every relative markdown link resolvable; the docs gate walks `docs`, `rules`, `skills`,
  `agents`, `context`, `memory`, `architecture`, plus the root `README.md` and `AGENTS.md`.
- MUST explain _why_ in prose and leave _what_ to the code: a comment that restates the next line is
  noise that will rot.
- MUST comment the non-obvious invariant at the place it is easy to break — why login sets
  `skipRetryOnUnauthorized`, why the console sink is the one place `console` is legal.
- MUST state a limitation honestly where one exists, as the secure-storage web fallback and the iOS
  verification script both do.

## Forbidden

- NEVER link to a file that does not exist yet, including a planned ADR — the gate fails on the
  broken link, not on the missing plan.
- NEVER hand-edit anything under `.ai/`: it is generated from this corpus and regenerated by
  `npm run knowledge:build`.
- NEVER duplicate a rule's text into a module README; link to the rule instead, so there is one
  place to change it.
- NEVER describe behaviour the code does not have — aspirational docs are worse than none.

## Rationale

This repository is designed to be read by agents as well as people, and both fail the same way: they
trust a confident document. Gating link integrity and rule-doc presence keeps the corpus navigable;
keeping the rule text in exactly one place keeps it correct. The duplication and contradiction
checks exist because a knowledge base that says two things has already lost its authority.

```ts
// src/packages/error-reporting/error-reporting.facade.ts
```


## Enforcement

| Mechanism                                                | Command                            |
| -------------------------------------------------------- | ---------------------------------- |
| Relative-link resolution + one doc per architecture rule | `npm run quality:docs`             |
| Module `README.md` presence                              | `npm run quality:architecture`     |
| Agent entrypoint documents                               | `npm run quality:agent-docs`       |
| Links across the canonical corpus                        | `npm run knowledge:links`          |
| Repeated 5-line blocks between documents                 | `npm run knowledge:duplicates`     |
| MUST/NEVER stated over the same predicate                | `npm run knowledge:contradictions` |
| Documents nothing links to                               | `npm run knowledge:orphans`        |
| `.ai/` regenerated from the current corpus               | `npm run knowledge:build`          |

Manual review where mechanical enforcement is impossible: truthfulness. Every gate above checks
structure — links resolve, files exist, blocks are unique. None of them can tell that a README
describes last quarter's design in fluent, well-linked prose.

## Definition of done

- [ ] The module README matches the surface and invariants the code actually has.
- [ ] New rules have a `docs/eslint/<rule>.md`; new links resolve.
- [ ] `npm run quality:docs` and the knowledge checks pass; `.ai/` was rebuilt, not edited.
